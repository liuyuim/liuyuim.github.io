  • 命令总结

    通过 Anaconda 管理 Python 包


    Anaconda集成了常用的扩展包，能够方便地对这些扩展包进行管理，比如安装和卸载包，这些操作都需要依赖conda。conda是一个在Windows、Mac OS和Linux上运行的开源软件包管理系统和环境管理系统，可以快速地安装、运行和更新软件包及其依赖项。在Windows系统下，用户可以打开Anaconda Prompt工具，然后在Anaconda Prompt命令
    注意：如果希望快速了解如何使用conda命令管理包，则可以在Anaconda"conda-h”或"conda-help"命令来查看帮助文档。
    【检测anaconda是否被安装】
    conda -- version
    conda4.5.4
    一旦发现有，就会返回其当前的版本号。
    【查看python版本】
    python -V
    Python 3.8.8
    【环境】
    查看环境
    conda env list 或 conda info –-envs
    创建新环境
    conda create -n 环境名
    conda create -n 环境名 python=版本号
    切换环境
    conda activate 环境名
    退出环境
    conda deactivate 环境名
    删除环境
    conda remove -n 环境名 --all
    导出环境
    conda env export > 文件名.yaml（.yml）
    导入环境
    conda env create -f 文件名.yaml -n NewEnvName（-n可选，默认用yml里定义的环境名，但不能和已有环境重名）
    【复制/克隆一个环境】
    通过克隆来复制一个环境。这儿将通过克隆old_envs来创建一个称为new_envs的副本。
    conda create -n new_envs --clone old_envs
    通过conda info –-envs来检查环境
    你现在应该可以看到一个环境列表有 new_envs, old_envs
    删除一个环境
    如果你不想要这个名为new_envs的环境，就按照如下方法移除该环境：
    conda remove -n new_envs --all
    为了确定这个名为flowers的环境已经被移除，输入以下命令：
    conda info -e
    new_envs 已经不再在你的环境列表里了，所以我们知道它被删除了。
    【清理环境tarball】
    移除索引缓存，锁定文件，未使用的缓存包和tarball。
    conda clean -a -all
    检查哪些包没有在包缓存中被硬依赖到其他地方，并删除它们
    conda clean -p --packages / conda clean -t --tarballs
    帮助
    conda clean -h
    【查看当前环境下包信息】
    使用“list"命令可以获取当前环境中已经安装的包信息，命令格式如下。
    conda list， conda list | findstr numpy
    【查找包】
    使用search命令可以查找可供安装的包，命令格式如下
    conda search --full-name 包的全名
    上述命令中，--full-name为精确査找的参数.后⾯紧跟的是包的全名。例如，查找全名为“python”的包有哪些版本可供安装，示例命令如下。conda search --full-name python
    【安装包】
    使用install命令可以安装包。如果希望在指定的环境中进行安装，则可以在install命令的后⾯显式地指定环境名称，命令格式如下。
    conda install --name env_name package_name
    上述命令中，env_name参数表示包安装的环境名称，package_name表示将要安装的包名称。
    例如，在Python 3环境中安装pandas包，示例命令如下。
    conda install --name python3 pandas
    如果要在当前的环境中安装包，则可以直接使用install命令进行安装，命令格式如下。
    conda install package_name
    指定版本
    conda install package=version1
    安装或更新包，例如
    conda install numpy=1.9.3
    conda update numpy=1.93
    【指定channel进行安装package】
    conda install --channel[-c] https://conda.anaconda.org/anaconda tensorflow-gpu=1.9.0
    【手动安装package】
    pip 安装本地包
    pip install ～/Downloads/package-name.whl
    conda 安装本地包
    conda install --use-local ~/Downloads/package-name.tar.bz2
    执行上述命令，会在当前的环境下安装package_name包。 若无法使用conda install命令进行安装时，则可以使用pip命令进行安装。值得一提的是，pip只是包管理器，它无法对环境进行管理，所以要想在指定的环境中使用pip安装包，需要先切换到指定环境中使用pip命令进行安装。pip命令格式如下。
    pip install package_name
    例如，使用pip命令安装名称为see的包，示例如下。pip install see
    【卸载包】
    如果要在指定的环境中卸载包，则可以在指定环境下使用remove命令进行移除，命令格式如下。
    conda remove --name env_name package_name
    例如，卸载Python3环境下的pandas包，示例命令如下。
    conda remove --name python3 pandas
    同样，如果要卸载当前环境中的包，可以直接使用remove命令进行卸载，命令格式如下。
    conda remove package_name
    【更新包】
    更新当前环境下所有的包，可使用如下命令完成。
    conda update --a11
    如果只想更新某个包或某些包，则直接在update命令的后面加上包名即可，多个包之间使用空格隔开，示例命令如下。
    conda update numpy
    conda update pandas numpy matplotlib

    7zip压缩到单独压缩包-不要用管理员


    for /d %%X in (*) do "c:\Program Files\7-Zip\7z.exe" a "%%X.7z" "%%X" -p123456 -mx=9
    goto start
    = 可以是多行文本，可以是命令= 可以包含重定向符号和其他特殊字符= 只要不包含 :start 这一行，就都是注释
    例：
    for /d %%X in (*) do "c:\Program Files\7-Zip\7z.exe" a "%%X.zip" "%%X" 压缩文件夹到独立压缩包，参考：
    https://www.zhihu.com/question/31567041/answer/145462690
    for /d %%X in (*) do "C:\Program Files\7-Zip\7z.exe" a "%%~dpnX.7z" "%%X" -p123456 -mhe -mx=0
    "%%~dpnX.zip" %%~ni 代表文件名 %%~xi代表文件扩展名
    -p 密码
    -mhe 加密存档头（-mhe开关），因此文件名将被加密。
    -mx=9 压缩等级，
    0 不压缩
    1 快速压缩
    5 正常压缩
    7 最大压缩
    9 极限压缩

    有/d：只将该目录下文件夹（不包括文件）压缩
    无/d：只将该目录下文件（不包括文件夹）压缩
    根目录已有目录结构：
    RootDir/
    ├── Dir-1/
    │ ├── Dir_1-File_1
    │ └── Dir_1-File_2
    │
    ├── Dir-2/
    │ ├── Dir_2-File_1
    │ └── Dir_2-File_2
    │
    ├── File_1
    ├── File_2
    └── 7zip.bat

    有"%%X"：单独压缩包只限于每个文件夹下的内容
    RootDir/
    ├── 根目录已有目录结构
    ├── TestDir-1.zip/
    │ ├── TestFile-1
    │ └── TestFile-2
    │
    │
    无"%%X"：单独压缩包不只限于每个文件夹下的内容，把根目录其他文件夹和文件也添加进来
    RootDir/
    ├── 根目录已有目录结构
    ├── Dir-1.zip/
    │ ├── Dir-1/
    │ │ ├── Dir_1-File_1
    │ │ ├── Dir_1-File_2
    │ │ └── ...
    │ ├── Dir-2/
    │ │ ├── Dir_2-File_1
    │ │ ├── Dir_2-File_2
    │ │ └── ...
    │ ├── File_1
    │ ├── File_2
    │ └── 7zip.bat
    │
    ├── Dir-2.zip/
    │ ├── Dir-1.zip/
    │ │ └── Dir-1.zip下已有目录结构
    │ ├── Dir-1/
    │ │ ├── Dir_1-File_1
    │ │ ├── Dir_1-File_2
    │ │ └── ...
    │ ├── Dir-2/
    │ │ ├── Dir_2-File_1
    │ │ ├── Dir_2-File_2
    │ │ └── ...
    │ ├── File_1
    │ ├── File_2
    │ └── 7zip.bat
    │
    │

    = 本方法通过在Windows批处理文件中调用7zip实现。
    = 如果你需要把文件打包成ZIP格式的话，把 "%%X.7z" 改成 "%%X.zip"
    = 该目录下除了我们要压缩的那些文件夹和批处理文件外，没有其他多余的文件或文件夹。
    = 核对压缩文件的完整性。选中新生成的那几个文件点击鼠标右键，在弹出的菜单中选择7zip，然后再弹出来的次级菜单中选择“测试文件”。测试完毕后如果报告说“未发现错误”，就可以把压缩好的那些压缩文件包复制到移动硬盘里边去了。
    = 如果7zip软件安装到了系统里的其他地方，请找到它的位置并在批处理文件代码中做出相应修改。
    = 请不要使用管理员权限运行该批处理文件，否则会把文件压缩到windows/system32目录下。双击批处理文件就好。

    使用FFmpeg批量处理视频


    for %%i in (*.mp4) do ffmpeg.exe -i "%%i" -r 15 -vf scale="iw/2:ih/2" "output\%%~ni.mp4"

    goto start
    = 可以是多行文本，可以是命令= 可以包含重定向符号和其他特殊字符= 只要不包含 :start 这一行，就都是注释
    参考：https://blog.csdn.net/luwieer/article/details/115438086
    进入解压后的文件夹，找到bin文件夹，然后将里面的三个exe文件悉数复制到你的Windows\system32\（这里懒得配环境变量了），如果命令中输出到子目录，则需要创建同名目录，例如output文件夹。
    例：
    for %%i in (*.mkv) do "ffmpeg目录\ffmpeg.exe" -i “%%i” -vcodec copy -acodec copy “%%~ni.mp4” 视频封装
    for %%a in (*.flv) do ffmpeg -i "%%~a" -vcodec copy -f mp4 "%%~na.mp4"
    %%~ni 代表文件名 %%~xi代表文件扩展名

    ffmpeg -i Desktop/input.mp4 -s 1920x1080 -b:v 1M -r 20 Desktop/output.mp4
    -fs 10MB : 表示文件大小最大值为10MB
    -r 20：表示帧率设置为 20fps 肉眼最低能接受15
    -b:v :指定视频的码率 1M：码率的值 1M 表示 1Mb/s
    -b:a : 指定音频的码率
    -s 1920x1080表示分辨率为1920x1080
    可以结合上面的命令一起来使用

    ffmpeg -i demo.mp4 -c:v libx265 -x265-params crf=18:preset=placebo output/demo_2.mp4
    crf是Constant Rate Factor的缩写，它的值越小，画质越高，占用的空间越大。它的可选项为0~51，默认为28.当crf在20以下的时候，就能实现视觉上的无损。
    此外，FFmpeg还为我们提供了一些预设选项，我们可以通过preset选项，快速设置x265编码的时候使用到的参数。
    preset的可选项按照编码速度从快到慢，排列如下:
    ultrafast、superfast、veryfast、faster、fast、medium、slow、slower、veryslow、placebo
    编码速度越慢，则压缩效果及画质越好。preset选项的默认参数为medium。

    ffmpeg -i input.avi -filter:v scale=720:-1 -c:a copy output.mkv
    如果想要限制画面宽度，而对于画面高度想要自动的进行裁切，那么就要用到过滤器。执行下面一行指令。
    -1将告诉ffmpeg自动选择与提供的宽度相关的正确高度，以保持宽高比。如果用户给定了高度，那么-1则可以用于指定保持宽高比中的宽度。
    -s：是 --scale 的短选项，用于指定输出文件的画面宽高比；
    -c:a：是输出音视频的盒子，copy 说明了是不做任何处理，原样输出。

    ffmpeg -i input.avi -vf scale="iw/1:ih/2" output.avi
    比如下面这样对画面进行宽高50%的收缩。你应该会注意到我们使用了两个变量。
    iw：输入文件的画面宽度，ih：输入文件的画面高度，这两个是系统的常量，可以拿来直接使用。

    视频速度调整取决于setpts视频滤波器的数值，支持的范围是0.25-4，数值越大，速度越快，两倍速是0.5，四倍速是0.25，以此类推
    ffmpeg -i in.mp4 -filter:v "setpts=0.5*PTS" out.mp4
    ffmpeg -i B_in.mp4 -filter:v "setpts=0.5*PTS" B_out.mp4
    ffmpeg -i A4.mp4 -filter:v "setpts=0.25*PTS,setpts=0.25*PTS" out.mp4
    由于atempo只支持0.5到2.0的数值，如果想要调整只四倍速，可以填写两组atempo
    ffmpeg -i test.mp4 -filter：a "atempo=2.0,atempo=2.0" -vn test-2.mp4
    视频音频同时调整
    ffmpeg -i test.mp4 -filter_complex "[0:v]setpts=0.5*PTS[v];[0:a]atempo=2.0[a]" -map "[v]" -map "[a]" test-2.mp4
